# 数据处理流程说明

## 一、San Jose Pipeline（`san_jose_pipeline/`）

### 1. 数据来源

使用 SafeGraph 采集的 POI（兴趣点）访客数据，包含 San Jose 市在三个时间节点的快照：

| 文件 | 时间节点 | 含义 |
|------|----------|------|
| CA_SanJose_08-14.csv | 08-14 | 灾前（before） |
| CA_SanJose_08-21.csv | 08-21 | 灾中（during） |
| CA_SanJose_08-28.csv | 08-28 | 灾后（after） |

每条记录对应一个 POI（placekey），包含该 POI 所在的 CBG（poi_cbg）、访客来源 CBG 的分布（visitor_home_cbgs）以及经州级缩放校正的访问量（normalized_visits_by_state_scaling）。

### 2. 处理流程

整个 Pipeline 分为三个阶段：`SafeGraph CSV → Aij → pk / px`。

#### Phase 1：计算 CBG-CBG 交互矩阵 Aij

目标：将 POI 级别的访客数据聚合为 CBG 级别的交互矩阵。

处理步骤：

1. **筛选城市**：从全量 CSV 中筛选 `city == "San Jose"` 的行，获取该市涉及的所有 CBG（约 632 个）。
2. **CBG → POI 映射**：按 `poi_cbg` 列建立每个 CBG 包含哪些 POI 的映射。注意这里使用全量 df 而非城市筛选后的 df，因为同一 CBG 内的 POI 可能被标记为不同城市名称。
3. **CBG 编码对齐**：SafeGraph 中 `visitor_home_cbgs` 字段的 key 为 12 位 FIPS 码（前缀补零），而 `poi_cbg` 为浮点数形式。因此将所有 CBG 编码统一转换为 12 位零填充字符串，并构建索引字典以加速后续查找。
4. **POI 访客数据预处理**：对每个 POI，解析其 `visitor_home_cbgs`（JSON 字符串，格式为 `{home_cbg: visit_count}`），并用 `normalized_visits_by_state_scaling / total_visits` 计算缩放因子（scaling），对每个来源 CBG 的访问量进行校正。
5. **矩阵累加**：对每个 CBG，遍历其包含的所有 POI，将每个 POI 的访客按来源 CBG 归属累加到矩阵对应位置，最终得到 Aij 矩阵。Aij[i][j] 表示从 CBG j 流入 CBG i 的加权访问量总和。

输出：`Aij_CA_San Jose_{date}.json`，格式为 `{cbg_key: [访问量列表]}`。

#### Phase 2：计算 pk（度分布）和 px（净流入分布）

**pk（度分布）：**

1. 将 Aij 矩阵二值化（非零元素置 1）。
2. 对每一行求和，得到每个 CBG 的出度（即该 CBG 向多少个其他 CBG 有流出访问）。
3. 对出度值进行频次统计，以宽度为 2 的 bin 分组（共 150 个 bin，范围 1~300），每个 bin 内取频次的均值。
4. 输出为 150 个 (x, y) 数据点。

**px（净流入分布）：**

1. 分别计算 Aij 矩阵的行和（出度）与列和（入度）。
2. 净流入 = 入度 - 出度，正值表示该 CBG 为净流入目的地，负值表示净流出。
3. 对净流入值进行频次统计，以宽度为 20000 的 bin 分组（共 50 个 bin，范围 -490000~490000），每个 bin 内取频次之和。
4. 输出为 50 个 (x, y) 数据点。

#### Phase 3：格式化输出

**pk 缩放：** 将 pk 的 y 值除以缩放因子（灾前/灾后为 0.8，灾中为 0.6），使三个时期的数据可在同一坐标系下对比。

**px 归一化：** 将 px 的 x 值除以 400000（灾前/灾后）或 500000（灾中），y 值除以总频次后乘以 18（灾前/灾后）或 20（灾中），使数据归一化到适合绑图的尺度。

输出文件：

| 文件 | 内容 |
|------|------|
| pk_data_before.csv | 灾前度分布（缩放后） |
| pk_data_during.csv | 灾中度分布（缩放后） |
| pk_data_after.csv | 灾后度分布（缩放后） |
| px_data_before.csv | 灾前净流入分布（归一化后） |
| px_data_during.csv | 灾中净流入分布（归一化后） |
| px_data_after.csv | 灾后净流入分布（归一化后） |

## 二、Fig 4 绑图（`fig4-plot-code/`）

### Fig 4.1 与 Fig 4.2：理论线图

这两个子图分别绘制散点数据（含误差棒）与理论预测曲线的对比如下。绑图采用双对数坐标（log-log）。数据点来自外部计算结果，理论线由 `theory.csv` 提供。

- Fig 4.1：x 轴范围 1~20，y 轴范围 0.1~1，展示数据与理论模型的吻合程度。
- Fig 4.2：x 轴范围 0.29~1，y 轴范围 1.9~15，尺度与 Fig 4.1 不同。

### Fig 4.3：城市相图（Phase Diagram）

这是核心图，展示多个城市在 β（回旋度/Gyration）- γ（互作度/Interaction）二维参数空间中的分布。

**数据输入：**

- `theory1.csv`：理论曲线 1（Moderate Disaster 阈值线 T1）
- `theory2.csv`：理论曲线 2（Severe Disaster 阈值线 T2）
- `PhaseDiagram.xls`：每个城市的 β、γ 坐标和城市名称
- `disaster-fig4.3-population.xlsx`：城市人口数据，用于缩放数据点大小

**数据点分组规则：**

首先按 β 值分为两大组：

- **Region D**（β > 2）：蓝色方块，代表高回旋度城市（天然恢复力强）
- **β ≤ 2 的城市**：进入下一步细分

对于 β ≤ 2 的城市，通过线性插值（`np.interp`）计算每条理论曲线在该 β 处的 γ 阈值（T1、T2），然后比较城市的实际 γ 值与阈值：

| 条件 | 区域 | 形状 | 颜色 | 含义 |
|------|------|------|------|------|
| γ < T1 | Region A | 倒三角 (v) | 红色 (#E8525F) | 脆弱城市（Collapse） |
| T1 ≤ γ < T2 | Region B | 圆形 (o) | 橙色 (#F7A83C) | 亚稳态城市（Conditional Recovery） |
| γ ≥ T2 | Region C | 正三角 (^) | 青色 (#4C929E) | 恢复力城市（Recovery） |

**数据点大小：** 按城市人口线性映射，人口越大，数据点越大。

**区域填充：** 在两条理论曲线之间用半透明色块标识三个区域：
- theory1 下方：粉色（Collapse 区域）
- theory1 与 theory2 之间：浅绿色（Conditional Recovery 区域）
- theory2 上方：浅青色（Recovery 区域）

**城市标签：** 每个数据点旁标注城市名，使用 `adjustText` 库自动调整位置以避免标签重叠。

输出为 SVG 和 PNG 两种格式。
